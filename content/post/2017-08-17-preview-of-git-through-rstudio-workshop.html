In a previous post I mentioned a future workshop: Reproducibility: Data Management, git, and RStudio. This workshop is the second part of a five-part R learning series. The five hands-on sections include:

  1. Intro to R: Data Transformations, Analysis, and Data Structures
  2. Reproducibility: Data Management, Git, & RStudio
  3. Visualization in R using ggplot2
  4. Mapping with R
  5. Developing Interactive Websites with R and Shiny

The goal of the second workshop is to leverage RStudio’s Git orchestration features. In the workshop I will interactively demonstrate reproducible report generation. Reproducibility is a critical outcome of Data Management and using Git’s version control is a practical way to implement a more ideal workflow. This workshop is aimed at Git beginners because, especially if you have only dabbled with Git, you may have noticed Git commands feel can feel arcane. I really like RStudio’s simplified Git controls. By transforming the main Git function into clickable buttons, RStudio reduces the need to memorize esoteric commands. This allows researchers to focus on workflow and productivity without sacrificing version control or collaboration.

The workshop will offer a gentle introduction to Git via RStudio. Users will gain hands-on, practical experience creating and manipulating a repository. Attendees will arrive with their own laptop and be guided past the obscure Git configuration process. Upon completion, users will have configured their own laptop and be able to deposit work-versions to Duke’s GitLab instance. The researcher will now be poised for better version control through Git repositories: able to collaborate, or simply exercise public and private repository control.

The workshop is open to the Duke University Community. A recording will be made available for self-paced instruction following the in-person presentation. I’m posting my resources for my portion of the workshop: Using RStudio to orchestrate Git – currently in draft.
