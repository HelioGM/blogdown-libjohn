In a previous post I mentioned a future workshop: Reproducibility: Data Management, git, RStudio. This is the second in a five-part series of practical, hands-on workshops making up our R learning series. The five parts include:

  1. Intro to R: Data Transformations, Analysis, and Data Structures
  2. Reproducibility: Data Management, Git, & RStudio
  3. Visualization in R using ggplot2
  4. Mapping with R
  5. Developing Interactive Websites with R and Shiny

The goal of the second workshop is to leverage RStudio’s Git orchesteration features. We will demonstrate and using R for reproducible data science and report generation. We believe a critical component of that reproducibility is Git’s version control and we really like how RStudio has simplified Git’s function by reducing the need to memorize git archane commands.

To that end we will provide a gentle introduction to promote the use of Git in developing report outputs through R. Through this hands-on workshop users will gain practical experience foo foo foo, hands-on workshop where users unfamiliar with Git will come with a configured laptop. away having configured their laptop to Duke’s GitLab instance;

The workshop is open to the Duke University Community (registration link, below.) The workshop will be recorded and made available for self-paced instruction after the presentation. The recording will be made available publiclly so there is no need to register if you’re not part of the Duke Community.1
