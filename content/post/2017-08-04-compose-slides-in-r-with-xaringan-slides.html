Last week I was compelled to write about the eternal verities of R Markdown. This post entails one of the many awesome report formats – slides – enabled by R Markdown. The ability to integrate slides as output of your analysis is a critical practical step in implementing reproducible data science.

Recently I began composing my presentation slides with a new R package, Xaringan – (pronounced Share-in-gan). Authoring is easy with Xaringan because slides are composed in R Markdown. Xaringan is optimized for RStudio so it’s easy to maintain versioning and sharing via github. Slide output is actually HTML so the the slides are easily portable on a flash drive, hosted on a laptop for presentation, or even served via any webhost (no more wondering if I have a back-up version of my slides.)

Since the editor pane and authoring method is well integrated into R and RStudio, inserting analysis and visualization generated from R code is amazingly simple. As such, analysis and code are part of the slide authoring process. This integration helps maintain continuity between analysis and the final report. Stock images are easy to insert as are any standard slide objects, all of which can be inserted with ease just like any presenter tool.

“If you don’t have a mouse, can you even present?” –John 226:Bostock

Visually the default slide style is spare, almost austere, but given how most slides are too cluttered (IMHO), I find this an advantage. The installed Xaringan package comes with a standard CSS stylesheet, which can be customized to suite personal needs. The whole slide life cycle (analysis, authoring, storing, and archiving methods) is very much inline with a tidyverse approach, thus bringing a nimble approach to reproducibility while enabling you to document your slides with literate code practices. Notably, you do not have to be a coder to take advantage of this approach. Xaringan slides can be authored with the same ease as with any other normal or more proprietary tools. Just spend 20 seconds learning R Markdown ADD LINK and you’ll be ready to increase your RQ (“Reproducibility Quotient”) in no time.

Last points. I suspect Xaringan Slides when used within the RStudio IDE is far more satisfying than MS-DOS for PowerPoint was. But the two systems must have striking similarities in the way that UNIX has influenced so much of modern computing. And if you’re wondering, my mouse remains a vital and relevant partner in all my quotidian slide production rituals. Thanks be to Engelbart3.
